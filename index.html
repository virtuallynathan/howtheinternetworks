Welcome to the Internet

This website is designed to teach you about how the internet works. We will follow the path of a packet from your home to one of the internet hubs and beyond.

To begin learning, just scroll down!

Home Gateway

For many computer users, the classic Linksys blue home gateway may be what they think of as "The Internet", so lets start there. This wireless home gateway is connected to a modem or other device via Ethernet. It then uses an internal ethernet switch and wireless modem to provide access to multiple clients simultaneously. The router makes use of NAT, or network address translation to multiplex TCP and UDP packets while using only a single externally reachable IP address.

A typical modern home gateway contains five 1GbE ethernet ports - one uplink and four switchports, and supports 802.11ac the latest high-bandwidth wireless standard supporting up to 1.3Gbps of wireless throughput. See CNET Router Recommendations for more information on current routers and Home Gateways.

Modem

The modem acts as a media converter between, in this case, a coaxial cable carrying DOCSIS (Data over cable service interface specification) data using quadrature amplitude modulation (QAM) and Ethernet. This could just as well be a DSL, or digital subscriber line connection over twisted pair cable, which carries data using discrete multitone modulation (DMT), or an ONT, optical network terminal, which carries data using time division multiplexing (TDM) and asynchronous transfer mode (ATM). The frequencies are de-multiplexed and demodulated and converted into Ethernet frames in the modem. In some cases, the modem will have a router integrated.

Modern DOCSIS modems are capable of up to 900Mbps using 24 channels, while DSL modems are limited to around 50Mbps.

Subscriber line termination

The modem communicates back to a central office or telephone exchange, which contains equipment to create the signals that travel over the coaxial, twisted pair, or fiber optic cabling. The item pictured is a cable modem termination system, which converts ethernet frames to DOCSIS frames to then be sent to a particular cable modem. A DSLAM, or Digital subscriber line access multiplexer operates in a similar fashion. Between the subscriber and the CMTS or DSLAM there may be other aggregation devices such as fiber nodes which convert the signals from coaxial cable to fiber optical cables for longer distance transmission. Each CMTS may aggregate and manage the connections of over 20,000 cable modems. This device has 1 or more ethernet interfaces, generally fiber optic, to uplink to a router.

The device pictured, an Arris E6000 CMTS, has sixteen 10GbE SFP+ (Small form factor pluggable optics) connections, across 2 routing linecards. It supports up to 128 downstream channels (38Mbps each) or up to 24 upstream channels (30Mbps each) in each of the 12 CAM (cable access module) slots.

Aggregation Switch

In large cities, many CMTSs or DSLAMs may be required to provide service to a neighborhood or area. If all of these subscriber termination units reside in the same headend or telephone exchange, the operator may wish to aggregate the bandwidth utilized by this hardware using cheaper switches before connecting it to a more expensive router port. This switch provides the internet uplinks to these devices, and the subscribers. This allows the network operator to increase efficiency by aggregating capacity at the edge of the network.

The device pictured is a Juniper Networks EX4500, it has forty 10GbE SFP+ (Small form factor pluggable optics) connections, and two 40GbE QSFP+ (quad SFP+) uplink connections.

Edge Router

Multiple aggregation switches can be connected to an edge router, and is likely the first device to perform any real routing function. Some routing may be run on the aggregation switch, but not necessarily. This device can aggregate multiple 10GbE or 40GbE connections to the core of the network. This router would likely run an interior gateway protocol (IGP) such as IS-IS, OSPF, or iBGP. Some business customers may be connected to the edge router via an exterior gateway protocol (EGP), such as BGP.

The router pictured is a Juniper MX480, it has 6 line card slots with up to 32 10GbE SFP+, or 2 100GbE CFP plus 4 10GbE SPF+ connections per line card for a total of up to 192 10GbE ports or 12 100GbE and 24 10GbE ports.

This Edge router may be part of a Regional Area Network (RAN) or Metro Area Network (MAN), consisting of multiple central offices or telephone exchanges.

Core Router

A number of Regional or Metro networks will be aggregated at a Core routing site. These sites are strategically chosen at locations where many other networks have a presence. Core routers are extremely powerful, and contain many line cards. Some core routers are "multichassis", including the one pictured, meaning that many chassis can be connected together to act as a more powerful single router. The router pictured is a Cisco CRS-X (Carrier Routing System). One the left is the CRS-X 16 Slot chassis, followed by the 24 slot fabric chassis (for building a multi-chassis system), and on the right there are 2 CRS-X 8 slot chassis. The CRS-X has up to 4 100GbE CFP connections per linecard, for a total of 64 100GbE connections per chassis. To connect to another router in another part of the country or world, the ports are connected to a packet optical transport system to carrier the data as a wave on a fiber optical cable.

Core routers are generally placed at strategic places around the globe, and in the case of large, global networks, this includes cities such as: London, Amsterdam, Frankfurt, New York, Ashburn, and Chicago, to name a few. This will be covered in more detail later. Read about Internet Exchanges

Packet Optical Transport

The core routers in a network are connected using optical transport, which supports aggregating multiple 10, 40, or 100GbE waves onto a single fiber optical cable. This is achieved using DWDM, or dense wave division multiplexing. Each wave on the ITU grid carries a 10, 40, or 100GbE channel, and there may be as many 96 waves on a single fiber. The hardware pictured is an Infinera DTN-X, which is capable of aggregating up to 250 10GbE connections on to a single fiber - a whopping 2.5Tbps! Using multiple chassis, over 24Tbps can be placed on a single optical fiber. The optical transport space has been advancing quickly, making use of Coherent optical technology, forward error correction, electronic dispersion compensation, and raman amplification to boost capacity and reach.

Amplifiers may be required every 50-100 km depending on the type of fiber optic cable used, and the modulation of the optical signals. DP-QPSK can travel farther than 16-QAM without amplification, but the maximum data rate is smaller due to the reduced spectral efficiency of the DP-QPSK coding. This allows providers to tune the optical reach and capacity to the optimum for their deployment - it may be possible to transport fewer channels over huge distances, in the thousands of kilometers.

Where is the Internet?

The internet is more of a physical than one might think. The Internet, at its core, is a collection of interconnected routers, and interconnected groups of routers called autonomous systems. These routers, and their interconnections are housed in only a buildings around the world. In this section we will dive into these locations.

Lets go!

Carrier Hotels

As mentioned above, networks chose to colocate their equipment inside of carrier-neutral facilities around the globe. This allows the carriers to sell their services out of that data center, as well as connect to many other networks. Many of these facilities are rather famous in the networking world for their density of unique internet service providers, or carriers. These buildings are aptly named Carrier Hotels". Equinix is a data center operator who runs a number of these carrier hotels. Equinix was founded during the dot-com boom, and grew rapidly until the bubble burst. After a few slow years, their growth continued and they are still one of the largest players in the data center industry.

Equinix operates one of the most popular Internet Exchange fabrics in the US, which will be discussed in detail later.

Internet Exchanges

Internet Exchanges were created as part of the natural growth of the internet. Internet service providers needed places to pass traffic between each other, and these locations were started in the early 1990s as NAPs or Network Access Points. One of the most famous was operated by MCI, MAE-East and MAE-West. These were quite large exchanges with over 80 members at peak. These exchanges were operated by ISPs, and were only really operated for business reasons. When it no longer made business sense to operate the exchanges, there were shut down.

In Europe, the model was slightly different, as the internet exchanges tend to be operated by non-profit organizations. The exchange pictured is the Amsterdam Internet Exchange or AMS-IX. It was founded in 1994 by CERN as a way to exchange scientific traffic, but allowed other carriers to connect as well. The AMS-IX organization was founded in 1997, and has been around since then. The exchange now sees peak traffic of over 2.5Tbps. Over 600 unique networks are connected to AMS-IX.

Other popular exchanges include LINX, the London Internet Exchange; DECIX, the Deutsch Commercial Internet Exchange; NETNOD, a swedish internet exchange; PLIX, the Polish Internet Exchange, and many others. An organization called Euro-IX has been founded to try and agree on policies across all of the european internet exchanges.

In the US, many exchanges are operated by data center providers, such as Equinix and Coresite. This is beginning to change, as network operators prefer the transparent, non-profit, community governed approach of the European internet exchanges like AMS-IX. Members of NANOG (The North American Network Operators Group) have formed a group called Open-IX dedicated to expanding the presence of european-style Internet Exchanges in North America. This has thus far been a successful effort: LINX has formed LINX NoVA, an Internet Exchange in Northern Virginia / Washington, DC, and both AMS-IX and DECIX have opened exchanges in the New York City metro area.

Who runs the Internet?

The internet, and the relationships between networks and companies are managed by surprisingly few people - the network engineers and peering coordinators of the networks that make up the tier 1 and 2 networks (and sometimes tier 3).

Lets go!

Relationships on the Internet

There are only a few types of relationships between networks on the internet: Peering (Settlement-free Interconnection), Paid Peering, and Transit.

Transit

Transit is essentially wholesale internet access, providing access to every other network on the internet. Paying for transit from a tier-1 provider will provide access to everything within the default free zone, or DFZ. At the current time this is over 500,000 routes.

Internet transit is often purchased on 1GbE and 10GbE ports, but sometimes on 40GbE and increasingly on 100GbE ports. Different amounts of bandwidth may be committed on each port, and generally different billing schemes are offered. A heavy-usage buyer may pay for a flat-rate 10GbE port in order to get the maximum capacity and value, whereas a smaller buyer may go for a 1Gbps commit on a 10GbE port with 95%ile billing. This company must pay for at least 1Gbps of bandwidth, the usage is sampled every 5 minutes for the course of the billing month and the highest 5% of the samples are discarded.

See Dr. Peering on Transit for more information.

Peering

Peering, or Settlement-free Interconnection, is when two networks agree that it is mutually beneficial for them to connect their networks without monetary exchange. This is generally the case if the amount of traffic being sent or received is roughly balanced, or if huge amounts of traffic are coming from one particular provider over a networks transit ports. This second situation is undesirable as you would essentially be paying the transit provider for traffic you could potentially receive for free.

See Dr. Peering on Peering for more information.

Tiers of the Internet

The Internet, at its core, consists of an area called the "Default Free Zone" - traffic between companies, know an Tier 1 internet service providers, is exchanged without monetary exchange, and these networks each have a complete BGP table and need no default route to send traffic. This collection of ISPs is rather small, consisting of: Level3 Communications, Verizon Business, AT&T, Sprint, Zayo, CenturyLink, GTT, XO Communications, Tata Communications, NTT Communications and TeliaSonera. These 11 internet service providers make up the core of the internet by peering freely at every mutual location. Because of this, most internet traffic crosses over one of these networks. All other internet service providers that wish to reach the rest of the internet must buy transit from one of these 11 carriers, or another provider who does. JANET, the UK educational research network purchases transit from Level3 Communications, TeliaSonera, and GTT.

While there may be no true "Global" Tier 1 ISPs, the main Tier 1 providers listed above have relationships with most or all of the regional Tier 1 ISPs for any given region. See Dr. Peering on Tier-1 ISPs for more information.

Types of Networks

Networks on The Internet are generally broken into 2 categories: Content creators / hosters, and content consumers. These are otherwise known, respectively, as Content Heavy networks, and Eyeball Networks.

Lets go!

Content Heavy Networks

The first type of network is a content delivery or content heavy network. These networks serve to deliver content to end users on behalf of other companies. For example, Apple uses Akamai to deliver iTunes content. Akamai is one of the largest CDN networks, representing over 20% of peak time global internet traffic. Other such CDNs include Limelight Networks, Level3, Highwinds, and Amazon. Other networks are not specifically hosting content for other companies, but may host many popular websites or servers - such networks include OVH, Leaseweb, and Amazon. According to research from Arbor Networks, and their Chief Scientist Craig Labovitz, networks he has dubbed "hypergiants" represent huge percentages of total inter-domain internet traffic.

Here is a photo of a Netflix CDN router cluster in Equinix Chicago: Netflix ASR9922 4Tbps router

Eyeball Networks

The second major network type is the Eyeball network. These networks are responsible for consuming huge percentages of the traffic served by content heavy networks and content delivery networks. These networks are generally home and business internet service providers, such as Comcast, who have millions or even tens of millions of internet customers. It is stated in the Sandvine global internet phononena report that over 50% of peak time internet traffic in the United States is from Netflix and YouTube alone.

Exploring the Internet

Traceroute is a neat tool included in most operating systems. It works by sending packets with an increasing time-to-live or TTL, and recording the IP of each hop it recieves an ICMP response from. This allows it to build a list of hops on the path to a destination host or IP address.

Lets go!

Traceroute

The traceroute to the left is from a server in New York to the St Andrews computer science network. As we can see, the path goes via Newark, New York, London, and Manchester (where JANET purchases 3 10GbE ports from Level3) before moving into the JANET6 network and going up to Leeds, Dundee, and finally back down to St Andrews. A keen observer might notice that the latency jumps from around 1ms to 76ms after the first hop - this is not because the latency between the 2nd and 3rd hop is huge, it is due to the fact that Level3 operates an MPLS core, and implements ICMP tunneling. This means that the ICMP packet makes it all the way to the edge of the Level3 network before a response is issued.

This network is connected to a Juniper router on the edge of Level3's network - xe is the abbreviation for 10GbE on a Juniper router. This edge router will run a full BGP table, and tag incoming packets with an MPLS label. MPLS, or multi-protocol label switching prevents each router on the network from having to contain the whole BGP routing table. It reduces the intelligence needed in the core of the network, by pushing it to the edge. Paths in the network are defined by Label Switch Paths (LSPs), which can be planned out using complicated planning software. This allows network operators to plan for failures and make sure traffic keeps flowing without interuption or congestion. The packets are carried over the network using multiple 10GbE or larger links, called aggregate ethernet or "ae".

More information can be found here, in Richard Steenbergen's NANOG presentation on traceroute
